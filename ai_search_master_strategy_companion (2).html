Repeated Structural Patterns

Analyzing the past papers reveal clear architectural templates. The exam graders use identical core engines with minor seed modifications. Master these five distinct question templates.

Pattern #1

Custom Grid Path-Tracing (Q25 - Q32 Template)

Every term includes a 10x10 custom coordinate map (connected by two-way paths) where students must output the exact trace and path for DFS, BFS, Best-First, and Hill Climbing.

Historical Evidence:
  • 2024T1 & 2024T2 Quiz1: Layout containing locations S, B, H, A, I, D, E, F, C, G on a coordinate grid. Starting node is $S$, Goal is $G$. Heuristics calculated using standard grid Manhattan distance.
  • 2024T3 Quiz1: Start $S$, Goal $G$ utilizing Manhattan distance, generating a strict alphabetical node traversal priority.
  • 2026T1 AI: Replaced the generic S-G map with a customized 4x3 chessboard Knight's Tour grid. Distances computed via Euclidean formulas (e.g., $d(G,D) = 3.6$), but retained the exact DFS/BFS/BFS/Hill Climbing structural questions sequentially!

Essential Formulas Cheat Sheet

Grid Heuristics

Manhattan Distance: d = |x₁ - x₂| + |y₁ - y₂|

Euclidean Distance: d = √((x₁ - x₂)² + (y₁ - y₂)²)

Savings Heuristic

Savings S(i,j): d(i,F) + d(j,F) - d(i,j)

*Where F is the fulcrum (base) node. Select highest valid S(i,j) without closing early loops.

Combinatorics (TSP)

Total Tours (Asym): (N-1)!

Total Tours (Sym): (N-1)! / 2

Stochastic Search

Move Probability: P = 1 / (1 + e^(-ΔE/T))

*As Temp (T) → ∞, P → 0.5 (random walk). As T → 0, P gets strict.

Trick #1: Strict Alphabetical Order Rules

The single biggest point loss on historical exams comes from expanding neighbors in the wrong sequence during blind searches.

How to handle children:
  1. Generate *all* valid immediate neighbors of the current parent node first.
  2. Sort them alphabetically immediately (e.g., if neighbors of D are B, A, E, they must be formatted as A, B, E).
  3. Run the RemoveSeen check *after* alphabetical sorting. Any node already present in either $OPEN$ or $CLOSED$ queues is dropped.
  4. Insert the surviving sorted neighbors into the $OPEN$ list based on the search type:
    • DFS: Prepend to the *front* of the list (preserving their alphabetical order, e.g., A first, then B, then E).
    • BFS: Append to the *back* of the list.

Trick #2: Path-to-Ordinal Conversion Shortcuts

Converting representations manually is error-prone. Use this visual subtraction trick:

Reference List: [A, B, C, D, E, F, G, H, I, J, K, L]
Task: Convert path E, A, D, ...
  1. Look at the first element E. In our reference list, it is at index 5 (using 1-based indexing: A=1, B=2, C=3, D=4, E=5). This is your first ordinal digit: 5.
  2. Delete E from the reference list. It becomes [A, B, C, D, F, G, H, I, J, K, L].
  3. Look at the next element A. In our modified reference list, it is at index 1. This is your second ordinal digit: 1.
  4. Delete A. The reference list is now [B, C, D, F, G, H, I, J, K, L].
  5. Next element is D. In our modified reference list, it is at index 3. This is your third ordinal digit: 3.

Repeat this step-by-step. The final digit is always 1.

Trick #3: Savings Heuristic Calculation

To compute savings when merging two nodes $i$ and $j$ around a base (fulcrum) node $F$:

The Core Formula:
$$S(i,j) = d(i, F) + d(j, F) - d(i,j)$$

Where $d(x,y)$ is the direct edge cost in the distance matrix.

Rule of Merging:

  • Sort the calculated savings values in **descending** order.
  • Choose the highest savings edge first.
  • Never select an edge that creates a closed loop (unless it contains all the cities, marking the final path completion).
  • Never select a node that is already connected to two other nodes in the active route (degree constraint).

Trick #4: Zero-Tolerance Grader Formatting

The IIT-M grading script uses strict exact string evaluations. Even if your math is correct, formatting errors can cost you the points.

CORRECT S,B,A,C,E,G
WRONG S, B, A, C, E, G (contains spaces)
WRONG S,B,A,C,E,G. (trailing period)
WRONG S-B-A-C-E-G (wrong separator)

Ensure you lock CAPS lock and strictly avoid using spaces or special symbols unless prompted.
